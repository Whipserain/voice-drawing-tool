# 纯语音控制绘图工具 — 设计文档

## 一、项目概述

### 1.1 设计目标
开发一款**纯语音控制**的绘图工具，用户不能使用鼠标或键盘，仅通过语音指令完成全部绘图创作。

### 1.2 核心挑战
| 挑战 | 解决策略 |
|------|----------|
| 语音识别准确性 | 模糊匹配 + 同义词映射 + 容错解析 |
| 响应延迟 | 命令即时执行 + TTS异步反馈 |
| 复杂指令拆解 | 自然语言分词 + 复合命令拆分 |
| 位置指定困难 | 9宫格区域定位系统 |
| 用户不知道说什么 | 启动引导 + TTS提示 + 帮助面板 |

## 四、技术架构

### 4.1 模块结构
```
├── SpeechRecognitionModule  — 语音识别（Web Speech API）
│   ├── 自动重启机制
│   ├── 错误恢复
│   └── 实时中间结果
├── SpeechSynthesisModule    — TTS语音反馈
│   ├── 队列管理
│   ├── 优先级中断
│   └── 中文语音选择
├── VoiceCommandParser       — 自然语言命令解析
│   ├── 文本预处理（中文数字、标点、同义词）
│   ├── 复合命令拆分
│   ├── 正则+关键词提取
│   └── 命令描述生成
├── CanvasDrawingModule      — 画布绘图
│   ├── 9宫格区域坐标系统
│   ├── 形状绘制（10种）
│   ├── 自由画笔模式
│   ├── 文字渲染
│   └── 历史记录（撤销/重做）
└── App                      — 主应用逻辑
    ├── 命令执行分发
    ├── 状态管理
    ├── UI反馈
    └── 欢迎引导
```

### 4.2 命令处理流程
```
用户语音输入
    ↓
SpeechRecognition (Web Speech API)
    ↓ 识别文本
VoiceCommandParser.parse()
    ↓ 预处理 → 拆解 → 解析
命令数组 [{action, shape, color, size, region}, ...]
    ↓ 逐个执行
App.executeCommand()
    ↓ 分发到对应方法
CanvasDrawingModule 绘制操作
    ↓ 同时
SpeechSynthesisModule 语音反馈
    ↓ 同时
UI视觉反馈（浮动卡片、状态栏更新）
```

### 4.3 位置系统设计

采用**9宫格区域定位**，而非像素坐标：

```
┌─────────┬─────────┬─────────┐
│  左上    │   上    │  右上    │
├─────────┼─────────┼─────────┤
│  左     │  中间   │   右    │
├─────────┼─────────┼─────────┤
│  左下    │   下    │  右下    │
└─────────┴─────────┴─────────┘
```

**设计理由**：
- 用户很难通过语音精确指定像素坐标
- 9宫格符合人类直觉的空间描述方式
- 每个区域有多个同义词（如"左上"="左上角"="左上方"）
- 区域内坐标自动计算，兼顾精度和易用性

## 五、关键技术决策

### 5.1 为什么选择 Web Speech API？
- 浏览器原生支持，无需安装
- 支持中文实时识别
- 免费，无API调用费用
- **局限**：仅Chromium内核浏览器支持

### 5.2 为什么不用外部NLP服务？
- 离线可用（语音识别本身需要网络，但命令解析不需要）
- 零成本
- 延迟最低（本地正则匹配，毫秒级）
- **局限**：无法理解非常自由的自然语言表达

### 5.3 为什么用TTS而不是纯文字反馈？
- 纯语音控制意味着用户可能不看屏幕
- 语音确认让用户知道操作已执行
- **局限**：TTS中文语音质量因浏览器而异

### 5.4 为什么移除了AI图片生成功能？
- 原版需要OpenAI API Key，增加使用门槛
- DALL-E调用延迟高（10-30秒），破坏语音交互的即时感
- 纯语音控制的绘图工具核心价值在于"即时绘制"

## 七、已知限制

1. **语音识别准确率**：Web Speech API的中文识别在嘈杂环境下可能不准确
2. **同音字问题**：如"圆"和"元"，通过上下文和同义词缓解
3. **复杂句式**：过于口语化的表达可能无法解析（如"给我来个红红的大太阳"）
4. **连续操作延迟**：快速连续说多个命令时，每个命令需要等待TTS播放完
5. **浏览器限制**：语音识别需要HTTPS或localhost环境
